# Licensing map — which file governs what

Enqpy™ is governed by four instruments, each covering a different thing. If you
only read one file, read this one first.

| What | Instrument | You may… |
|---|---|---|
| **The cipher** (invention / patents) | `COVENANT.md` — signed, irrevocable, binds successors | implement, port, and deploy at any scale, including commercially — free, no permission |
| **Proof, FCD, test vectors, guides** | `LICENSE-DOCS.md` — CC BY 4.0 (vectors CC0) | copy, mirror, translate, re-host anywhere, forever (keep attribution) |
| **Reference source code** | `LICENSE` — NQP LLC Public License Rev 2.0 (evaluation) | clone, build, run, study; **not** redistribute the source without agreement |
| **Names / marks** | Foundation Trademark Use Guidelines (and `LICENSE` §D) | refer to the marks factually; not imply endorsement or certification |

Related files: **`CITATION.cff`** (how to cite) · **`PROVENANCE.md`** (the
"if you found this" note and preservation map).

## Why it's split this way

The commercial model — certification, the Enqpy™ marks, proprietary
high-performance implementations, and services — funds the work and is unaffected
by any of the open grants above. The **cipher itself** is free forever under the
covenant, and the **documents that let anyone rebuild it** (the FCD and test
vectors) and **verify it** (the proof) are free to copy forever under CC. So the
science and the cipher survive independent of NQP LLC, the Foundation, or any
host — while the reference *source code* keeps its evaluation terms.

That separation is deliberate: **what must outlive everyone is freely copyable;
what funds the work stays governed.**
